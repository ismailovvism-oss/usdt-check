# Проверка прихода USDT (usdt-check)

Однофайловое веб-приложение: оператору обменника за секунды отличить настоящий
USDT от подделки и сверить сумму сделки прямо с телефона. Без бэкенда — чистый
клиент, работает в браузере. Файл: `usdt-check.html`.

## Стек и принципы
- Один файл `usdt-check.html`: vanilla HTML/CSS/JS, без сборки и без зависимостей.
- Данные берутся напрямую из TronGrid (CORS открыт):
  `GET https://api.trongrid.io/v1/accounts/{addr}/transactions/trc20?limit=N&only_to=true`
- Не добавлять фреймворки и npm-пакеты без явной просьбы. Если фичу можно сделать
  в одном файле — делать в одном файле.
- Адрес по умолчанию задаётся константой `DEFAULT_ADDRESS`; также читается из
  ссылки `#addr=` / `?addr=`. localStorage допустим (файл свой), но не обязателен.

## Критический инвариант — НЕ нарушать
- Настоящий USDT в сети TRON — ЕДИНСТВЕННЫЙ контракт:
  `TR7NHqjeKQxGTCi8q8ZY4pL8otSzgjLj6t` (константа `REAL_USDT`).
- Вердикт «Подлинно» допустим ТОЛЬКО если одновременно: `contract === REAL_USDT`
  И получатель === проверяемый адрес И (если задана) сумма совпала. Любое
  отклонение → «Внимание». Никогда не наоборот, никакого «мягкого» зелёного.
- Сумма = `value / 10^decimals`; `decimals` брать из `token_info` (у USDT = 6),
  не хардкодить.
- Газ-фри переводы: ориентироваться на событие перевода токена (контракт-эмитент),
  НЕ на внешний контракт-обёртку (GasFree Endpoint). Текущая логика это уже даёт —
  не сломать при рефакторинге.

## Чего приложение НЕ делает (и не должно молча обещать)
- Не проверяет заморозку/чёрный список и происхождение монет (AML-таинт).
- Не гарантирует необратимость (финальность) перевода.
- Поэтому в футере есть честная оговорка про сверку расходуемого баланса —
  сохранять её при любых правках UI.

## Дизайн
- Мобайл-фёрст, рассчитан на Samsung Z Fold5 (узкий и развёрнутый экран).
- Сигнатура — вердикт-«штамп»: крупная сумма + печать «Подлинно»/«Внимание».
  Цвет несёт смысл, а не декор.
- Шрифты: Space Grotesk (заголовки/вердикт), IBM Plex Mono (суммы/адреса/хэши).
- Уважать `prefers-reduced-motion`; видимый фокус для клавиатуры; контраст под
  использование при ярком свете.
- Тексты интерфейса по-русски, активным залогом, без системного жаргона.

## Дорожная карта (по одному пункту за раз)
1. Чёрный список Tether: проверять отправителя/получателя через `isBlackListed(address)`
   на контракте USDT (`triggerconstantcontract` в TronGrid). Приоритет №1 — для
   обменника важнее детектора подделок.
2. PWA: manifest + service worker, иконка, офлайн-кеш оболочки (данные онлайн).
3. История проверок в рамках сессии; копирование/экспорт отчёта.
4. (Опц.) сверка по хэшу транзакции, а не только по адресу.

## Рабочий процесс
- Правки атомарные. После изменений проверять, что JS валиден и страница
  открывается в браузере без ошибок в консоли.
- Перед изменением критической логики (классификация, инвариант) — сначала план,
  потом код.
